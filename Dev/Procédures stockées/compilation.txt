La compilation de la librairie levenstein 2.1.6 a été testée sous windows 32 et rhel 5


I. sous windows 32

	Prérequis
	-> Microsoft Platform SDK
	-> Microsoft visual Studio .net 2003 ou Microsoft Visual Studio C++ Express Edition

	Compilation :
	-> cl /I"Chemin vers include platform SDK" /I"Chemin vers include server postgresql" /I"Chemin vers include server\port\win32 postgreSQL" /I"Chemin vers include microsoft visual studio vc7" levenstein.c

		l'option /I doit être ajoutée avec le chemin vers les fichiers include

	puis Link :

	-> link.exe /LIBPATH:"Chemin vers lib microsoft visual studio vc7" /LIBPATH:"Chemin vers lib platform SDK" /LIBPATH:"C:\Program Files\PostgreSQL\8.2\lib" /def:def.txt /DLL /OUT:levenstein.so levenstein.obj

		L'option /DLL permet de créer une librairie
		L'option /OUT:levenstein.so spécifie le fichier en sortie
		L'option /def:def.txt permet d'exporter les méthodes présentes dans le fichier def.txt


II. sous RHEL 5

	-> gcc 44


	Compilation
	-> gcc -Ichemin vers include server postgres -Ichemin vers include /server/utils postgres -shared -o levenstein.so levenstein.c

		L'option -nostdlib permet de ne pas lier aux librairies standards
		L'option -shared permet de creér une librairie partagée (dynamique)
		L'option -o levenstein.so permet de spécifier le répertoire en sortie

		l'option -I doit être ajoutée avec le chemin vers les fichiers include
